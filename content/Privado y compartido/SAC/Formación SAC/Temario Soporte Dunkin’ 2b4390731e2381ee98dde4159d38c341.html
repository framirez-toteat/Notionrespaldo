    1. Problemas en una impresora en particular

    Se reporta un problema en una impresora en particular, en este caso se debe validar:

    ✅ 1. Verificar conexión física y red

    • Validar que la impresora cuente con papel y que no esté atascado
    • Confirmar que la impresora esté encendida y conectada por cable de red (no USB) al router o switch. (luz amarilla y verde)
    • Solicitar al cliente realizar el self-test de la impresora para validar IP y MAC.

    🧾 2. Cómo hacer el self-test de la impresora (modelo estándar)

    1. Apagar la impresora.
    1. Presionar y mantener el botón FEED.
    1. Encender la impresora sin soltar el botón FEED.
    1. Apenas salga el papel con la configuración, soltar el botón FEED.
    El papel debe mostrar la IP y MAC Address de la impresora.

    🟡 Si no aparece la IP/MAC:

    • Verificar que esté conectada por red, no por USB.
    • Si imprime papel en blanco: revisar si el papel está al revés.

    💻 3. Verificar IP del PC del cliente

    1. Pedir al cliente abrir el ejecutador: WIN + R y escribir: CMD ADMIN
    1. En la consola, escribir:
    • ipconfig
    Comparar IP del PC con la IP de la impresora.

    Ejemplo de diferentes segmentos (no hay conexión):

    • IP Impresora: 192.168.10.7
    • IP PC: 26.14.101.96

    Ejemplo de mismo segmento (debería haber conexión):

    • IP Impresora: 26.14.101.80
    • IP PC: 26.14.101.96

    🕵️‍♂️ 4. Verificar duplicidad de IP / detección de MAC

    1. Abrir CMD ADMIN nuevamente.
    1. Ejecutar el comando:
    • arp -a
    Revisar si la IP de la impresora aparece y si la MAC Address coincide.
    • Si la IP aparece con otra MAC ➤ conflicto de IP duplicada.
    • Si no aparece ➤ usar scanner de red como:

      Advanced IP Scanner

    🔌 5. Si no detecta la impresora, verificar físicamente

    • ¿Está conectada al router correcto?
    • ¿Está usando un puerto funcional del router o switch?
    • ¿Está el cable de red dañado?
    • ¿Está conectada a un switch apagado?
    • Cómo detener una impresora térmica que imprime infinitamente desde Toteat (error en Windows)

      ✅ 1. Abre los servicios de Windows

      • Presiona Windows + R en el teclado.
      • Escribe:
        services.msc
      • Presiona Enter.

      ✅ 2. Detén el servicio de impresión (Print Spooler)

      • En la ventana que se abre, busca el servicio llamado:

        Cola de impresión o Print Spooler.

      • Haz clic derecho sobre él.
      • Selecciona "Detener".

      ✅ 3. Borra manualmente los documentos atorados

      • Abre el Explorador de archivos de Windows.
      • Escribe en la barra de direcciones (o navega a):

      C:\Windows\System32\spool\PRINTERS

      • Si pide permisos de administrador, dale “Sí”.
      • Una vez dentro de esa carpeta, borra todos los archivos que veas.
      🔴 ¡Importante! Estos archivos son los documentos atorados que están haciendo que la impresora imprima sin parar.

      ✅ 4. Reinicia el servicio

      • Vuelve a la ventana de services.msc.
      • Haz clic derecho sobre Cola de impresión / Print Spooler.
      • Elige "Iniciar".

      ✅ 5. Verifica la impresora

      • Abre el Panel de Control > Dispositivos e impresoras.
      • Haz clic derecho sobre la impresora térmica (ej. POS80, XPrinter, EPSON).
      • Asegúrate de que no esté en “Pausa” ni marcada como “Usar sin conexión”.

      ✅ 6. Abre nuevamente el POS de Toteat

      • Reinicia el Print server de toteat si es necesario.
      • Ingresa al POS y envía una impresión de prueba

      💡 Consejo preventivo:

      Para evitar que se repita este problema:

      • Asegúrate de que solo haya una sesión del POS activa.
      • No asignes en TRM (toteat) impresoras que el TPS reconozca pero no estén creadas en Windows
      • Instala siempre el driver recomendado para la impresora térmica.

  • KDS de entrega

    El KDS es el sistema de apoyo de preparación y entrega de pedidos de los clientes. Si el sistema no está funcionando se debe validar:

    • Revisar que esté encendido
    • Validar que esté conectado a al red

    En caso que el problema persista se debe:

    1. Borrar caché del navegador
    1. Asegurarse que esté en la url correspondiente https://kitchendisplay.toteat.com/
    1. Iniciar sesión
    1. Seleccionar el local que corresponde, el sistema cerrará la sesión y se debe volver a ingresar.
    1. Al inicial sesión seleccionar Configuración KDS desde el menú latelar, y seleccionar Refrescar Listado y Activar este KDS en este Equipo
    1. Por último se seleccionar “Haga click aquí para cerrar sesión y usar KDS”
  • Funcionamiento tótem/kiosko

    Cuando un Totem deja de funcionar, se deben seguir los siguientes pasos

    1. Actualizar o refezcar el sistema con el logo de arriba a la izquierda que muestra el sistema
    1. Validad si el totem cuenta con conexión, para esto primero se debe desbloquear la pantalla, deslizando el dedo desde la parte inferior solo un poco. El sistema le pedirá ingresar una clave “2025”.
    1. Se debe realizar un speedtest en el navegador para validar que tenga buen internet.
    1. En caso que no sea problemas de internet, se debe proceder a resincronizar el Totem
      1. Borrar cache en el navegador
      1. Identificar correctamente el Totem con problemas.
      1. Al borrar caché en la url https://kiosk.toteat.shop/ se mostrará una pantalla que al seleccionar “Comenzar” entregará un código.

      d. Se debe ingresar a Toteat al ambiente que corresponde (local), e ingresar a la configuración de Kiosko de Autoatención

      e. Se debe ingresar a la sección “Dispositivos” y seleccionar “Editar” en el Totem que corresponde.

      f. Se abrirá una ventana donde en la parte inferior se encuentra “Sincronizar”. En esta sección se debe seleccionar “ingresar código” y se debe ingresar el código entregado en el Totem y seleccionar “Verificar”

      g. Una vez verificado se debe volver al Totem que seleccionar “Entrar”. De este modo quedará sincronizado el Totem nuevamente

      • No tenga restricciones o este Bloqueado.

    1. Actualizar o refrescar el sistema con el logo de arriba a la izquierda que muestra el sistema, se sugiere realizarlo 3 veces, igualmente se sugiere Reiniciar el Totem o borrar el cache.

  • ERROR Toma Inventario en Cierre de Turno

    A veces no es posible cerrar un turno porque la toma de inventario está pendiente. Es un error que puede darse por diferentes motivos:

    • Existen problemas de conexión a internet
    • El sistema queda pendiente de confirmación/guardado
    • No permite generar devoluciones asociadas al cierre

    Si no se logra realizar y confirmar la toma de inventario al momento del cierre, por configuración no podrá realizarse al día siguiente. Para solucionarlo hay que cambiar esa configuración y volver a activar una vez cerrado el turno.

    Sigue estos pasos:

    1. Menú Lateral - Restaurantes - Ajustes Generales - General - Validaciones adicionales cierre de turno.
    1. Dejar la opción en NO para proceder con el cierre, guardar cambios! y volver a dejar en SI luego del Cierre del turno.

    🧠 Consideraciones importantes

    • Esta incidencia es operativa, no de configuración permanente
    • La desactivación es temporal y controlada
    • Siempre reactivar la validación después del cierre
    • Reportar si el error se repite frecuentemente

  • Levantar TPS CMD (No es Reinstalable - Dunkin Cenit)

    En Dunkin Cenit Ocurre a veces que el TPS no se ejecuta correctamente y no levanta por mas que se reinicie, El TPS de este local funciona por CMD es distinto, por lo cual no se puede re instalar.

    Cuando lo reiniciamos dándole al Icono TPS, clic derecho Reiniciar se identifica que el TPS se Levanta y el CMD envía información pero NO procede a Imprimir, se observan el avión en Gris, y los pedidos continúan con icono de impresora en ROJO.

    Tener presente que todos los locales de Dunkin tienen las mismas 6 impresoras 4 Térmicas y 2 Totems.

    1. CAJA
    1. BEBIDA CALIENTE
    1. SALSERA
    1. VITRINA
    1. TOTEM1
    1. TOTEM2

    Por lo cual se sugiere Ingresar al PC Servidor y pausar Todas las impresoras (Mas que todo las 4 principales), para verificar que al hacer el proceso de CONFIG si impriman las comandas y boletas pendientes.

    SOLUCIÓN:

    Para hacer CONFIG de TPS Primero debemos MATAR el TPS, proceso normal de TASKILL en EJECUTAR, si no se Cierra, podemos hacerlo en Administrador de Tareas y Finalizar las tareas de todos los DRONE abiertos.

    Este proceso pueden realizarlo en otros locales de Dunkin sin problema tal cual, siempre y cuando no se levante el TPS como habitualmente.

    Una vez ya con TPS Abajo, debemos ir Configuración - Print Server Config

    Ingresar al Lápiz del TPS instalado

    En la Opción Copiar Configuración a Middleware, darle clic al icono negro de Texto, donde se desplegará la info para el config.

    Deberán seleccionar todo el texto interno y copiarlo para luego realizarlo en el pc servidor.

    Nos dirigimos a la carpeta de TPS dentro del PC Servidor de Dunkin y crearemos un nuevo archivo de DOCUMENTO DE TEXTO.

    Le pondremos de nombre Config

    Dentro de este archivo pegaremos el texto anterior del TPS, y debemos guardar cambios.

    Luego de esto debemos Proceder a Ejecutar el Drone como Administrador

    Deberá Ejecutarse el TPS con normalidad y se deberán ver los documentos de impresión en las colas de las impresoras y en TRM no deberán verse las mesas con el icono de IMPRESORA EN ROJO y el Avión ya pasará a estabilizarse en Verde con X.

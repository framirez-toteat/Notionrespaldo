Complemento:

  • Toteat Print Server

    CON INTERNET

    • TRM se comunica con GAE y se registran las transacciones
    • Print Server se entera que debe procesar documentos nuevos bien sea desde la nube o vía red local
    • Print Server recupera los nuevos datos desde la nube, los procesa, imprime y notifica de nuevo a la nube
    • TRM se entera de los cambios desde la nube

    ¿Qué es el modo offline?

    Es un modo de emergencia en el que opera el Print Server para actuar como servidor transaccional (en reemplazo de la nube) y asegurar de esta forma el funcionamiento de la operación en caso de intermitencia del Internet o caída de este.

    Este modo puede darse de forma forzada (por decisión de una persona) o de forma automática (cuando se detecta la caída desde la aplicación)

    Manual Offline (documento en Drive)

    SIN INTERNET

    • TRM se comunica con el Print Server vía red local y registra las transacciones

    Print Server procesa, imprime, actualiza y almacena los nuevos datos dentro de la máquina donde se está ejecutando

    • TRM se entera de los cambios desde el Print Server

    RECUPERANDO INTERNET

    • Ya sea porque se desactivó el offline forzado o se detectó el regreso del Internet en modo automático, Print Server busca sincronizar todos los nuevos datos generados a manera de que queden registrados en la nube. Solo si esta operación fue exitosa, entonces se sale del modo offline

  • Desactivar firewall

    Para acceder de forma rápida, usar los siguientes comandos:

    • Comando wf.msc
    • Comando netsh advfirewall set allprofiles state off

    o bien por:

    • Ingresar al Panel de Control → Sistema y Seguridad → Firewall de Windows Defender
    • Seleccionar Seguridad Avanzada → Propiedades de Firewall de Windows Defender e inactivar el Firewall en los tres perfiles (Dominio, privado y público).

    • Ejecutar el TPS descargado para instalar el TPS.

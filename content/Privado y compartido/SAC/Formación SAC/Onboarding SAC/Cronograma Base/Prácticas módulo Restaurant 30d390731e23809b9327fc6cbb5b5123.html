🎯 Objetivo del módulo

Que el agente comprenda la estructura base del sistema y cómo las configuraciones impactan la operación diaria del cliente.

🪜 Orden recomendado de prácticas

1️⃣ Turnos y Subturnos

🎯 Qué debe aprender

Cómo estructurar horarios operativos.

🧪 Prácticas

  • Crear varios turnos (ej: mañana, tarde, noche).
  • Crear sub turnos dentro de un turno.
  • Editar y eliminar turnos para ver errores.

🔎 Errores a provocar

  • Intentar eliminar un turno con sub turnos activos.
  • Superposición de horarios.

2️⃣ Sectores

🎯 Qué debe aprender

Organización del salón.

🧪 Prácticas

  • Crear sectores individuales.
  • Crear sectores masivos.
  • Crear subturnos.

🔎 Errores a provocar

  • Eliminar sector con mesas asignadas.
  • Modificar ID de sectores.

3️⃣ Mesas

🎯 Qué debe aprender

Asignación y estructura del salón.

🧪 Prácticas

  • Crear mesas individuales.
  • Crear mesas masivas.
  • Asignar mesas a sectores.
  • Desasignar mesas.

🔎 Errores a provocar

  • Eliminar mesas asignadas a sectores.
  • Modificar nombres y ID de mesas asignadas.

4️⃣ Cajas

🎯 Qué debe aprender

Configuración de flujo de dinero.

🧪 Prácticas

  • Crear cajas.
  • Revisar cada apartado de configuración.
  • Revisar qué modificaciones se les pueden hacer a las cajas abiertas y qué no.
  • Conocer los tipos de cierres.

🔎 Enfoque de aprendizaje

Comprender para qué sirve cada sección:

  • Apertura.
  • Cierre.
  • Control de movimientos.

5️⃣ Personal y usuarios

🎯 Qué debe aprender

Gestión de usuarios y permisos.

🧪 Prácticas

  • Crear personal individual.
  • Creación masiva.
  • Usar correo personal para pruebas.
  • Aceptar invitación de usuario.

🔐 Ejercicios de permisos

  • Cambiar perfiles.
  • Probar permisos según cada rol.
  • Identificar qué puede y no puede hacer cada perfil.

🔎 Errores a provocar

  • Usuario sin aceptar invitación.
  • Permisos insuficientes.
  • Errores en carga masiva de perfiles.

6️⃣ Clientes

🎯 Qué debe aprender

Creación y uso de clientes en órdenes y delivery.

🧪 Prácticas

  • Crear clientes para asignar a orden.
  • Crear clientes para delivery.
  • Crear clientes para documentos fiscales.

🌎 Validaciones importantes

  • Ejemplos de creación por país.
  • Datos necesarios para documento fiscal.

🔎 Errores a provocar

  • Cliente con datos incompletos.
  • Formatos incorrectos para documento.

🧠 Lógica del orden (por qué así)

Este orden permite que el agente entienda la operación real:

  1. Horarios → Turnos
  1. Espacio → Sectores y mesas
  1. Flujo de dinero → Cajas
  1. Personas → Usuarios
  1. Clientes → Facturación y delivery

Es exactamente como funciona un restaurante en la vida real.

✅ Resultado esperado del módulo

Al finalizar, el agente podrá:

✔ Comprender la estructura operativa del cliente.

✔ Detectar errores comunes de configuración.

✔ Explicar al cliente cómo organizar su restaurante.

✔ Identificar causas raíz en problemas de operación.

Introducción

Como aclaración inicial, es importante destacar que parte importante de los incidentes que suceden en el chat se debe al TPS (Toteat Print Server), el cual está diseñado para generar una conexión entre nuestros servicios y el local, lo cual permite tanto la impresión como gran parte de las opciones que usamos para los locales.

Mencionado lo anterior, es fundamental en gran parte de las operaciones que este programa se encuentre encendido y correctamente conectado. Para hacer una revisión inicial, tiene dos puntos donde podrás confirmar la conexión: 1. En la parte superior izquierda podrás encontrar un “avión” en verde y 2. En la parte inferior izquierda (iconos ocultos) el icono del programa en color verde.

Con estos dos aspectos visuales tendrás parte de la información del estado del TPS y su conexión principal en el local. Eso sí, pueden existir diferentes situaciones que pueden afectar la impresión y la conexión del TPS.

  1. Permisos o bloqueos del TPS: Al tener un programa que necesita conexión con nuestro servidor, puede suceder que el TPS tenga algunas restricciones desde el PC; para esto puedes revisar varias opciones.
    • Bloqueo desde el firewall:

      Como parte de la conexión se genera constantemente, el firewall del pc, puede crear faltas de conexión o que el TPS, no se ejecute correctamente, para esto podrás revisar en las configuraciones del firewall (para ingresar puedes usar el comando Wf.msc) y debe registrarte de esta forma, confirmando que el firewall no genere alguna restricción.

    • Permisos para TPS:

      Además de la configuración que se debe realizar dentro del firewall, es importante que la carpeta donde se localiza el programa tenga permisos totales en el PC, esto con el fin de que siempre se ejecute correctamente y la conexión se mantenga lo más constante posible.

      Para esto, debes ingresar a la carpeta raíz del PC y buscar la carpeta del TPS. Una vez allí, podrás ingresar a las propiedades de la carpeta y, en la opción de “Seguridad”, editar los permisos de los usuarios para que la carpeta tenga control total de uso.

  1. Conexión de red o intermitencia: Así como los permisos de la configuración, una conexión estable a la red, mejora el funcionamiento del TPS, en gran medida, por eso la intermitencia en la red o alguna caída temporal en la red, pueden afectar. Para esto pueden suceder dos situaciones:
    • Intermitencia en la red:

      Parte de la revisión que podemos hacer es directamente en nuestro programa de conexión con el cliente (ScreenConnect) con el cual podemos ver la conexión del cliente y si es estable o no

Una vez identificado el problema o situación que pudiera llegar a tener el cliente, podrás realizar diferentes procesos para reiniciar la conexión del TPS, y así recuperar la conexión para esto iremos de lo más sencillo posible a lo más complejo.

  1. Reinicio del TPS: Para reiniciar el TPS, tienes varias opciones usadas por el equipo para generar la conexión nuevamente:
    • Reinicio en el acceso directo:

      Para este reinicio, únicamente debe ingresar al ícono del TPS, en donde encontrarás la opción de “Restart” para hacer un reinicio sencillo o rápido del mismo

    • Reinicio desde la carpeta raíz:

      En caso de que el reinicio anterior no genere la conexión, se puede realizar el reinicio desde la carpeta raíz, para esto se debe ingresar al almacenamiento local del pc (carpeta C:/), y desde allí ingresar a la carpeta del TPS. Luego, ingresar a la carpeta “Data”, y desde allí generar la eliminación de varios archivos, siempre dejando los archivos “drone” y “ops” en la carpeta; los demás podras eliminarlos

      Una vez realizado este proceso, podrás ingresar a la carpeta del TPS, y buscar el programa “drone”, para ejecutar el TPS y esperar a que nuevamente el programa se muestre en la barra de tareas.

    • Reinicio desde cmd:

      Como tercera opción, puedes abrir el cmd y desde allí generar el reinicio de todo el TPS; para esto debes ingresar a la opción de “Ejecutar” (con el atajo en el teclado “Windows + R”), y desde allí escribir el código “cmd admin” y luego en la opción “aceptar”.

      Una vez que abras la aplicación, te mostrará un recuadro en negro para teclear algún código o información; desde allí debes colocar el código completo para reiniciar TPS. Esto automáticamente eliminará la información anterior de los archivos y abrirá automáticamente el TPS nuevamente.

  1. Reinstalar el TPS: Esto se usa en caso extremo de que no cargue o el programa no se ejecute, para esto se debe revisar que el PC tenga configuradas las impresoras que corresponden y podrás hacer el siguiente proceso:
    • Eliminación de la carpeta raíz:

      Para iniciar la eliminación, debes confirmar que el programa no se esté ejecutando; para esto puedes revisar desde el “Administrador de tareas” y desde allí buscar el proceso “drone”. En caso de que esté abierto, debes generar la finalización del proceso.

      Esto permitirá generar el proceso de eliminación de la carpeta raíz encontrada en el disco local (C:) y desde allí renombrar la carpeta para tener el registro del historia del uso (o logs del programa) y posibles errores, y que no interfiera con la instalación del nuevo programa. Una vez renombrada la carpeta, podrás realizar el proceso de instalación del TPS.

Conexión en offline:

Como entendemos que conectarse a una red no siempre es estable, para esto se creó el estado de Offline en el TPS. Con la intención de que estas caídas se pudieran controlar lo mejor posible y así mitigar o reducir el impacto por la falta de red, este estado se activa una vez el TPS tiene la configuración previa y la conexión en el PC servidor se cae o tiene intermitencia.

Errores comunes

  • Errores más comunes del paso a paso en la revisión del TPS

    1. Fatal error detected: Este error se presenta una vez instalado el TPS, justo antes de ejecutarse correctamente. Este error se puede dar por alguna falla en el servidor, para casos excepcionales. Regularmente este error está referido a algún problema en el proceso de implementación (instalación) del TPS, para solventarlo, debe desinstalar el TPS, borrar las descargas del TPS bajadas previamente (en las descargas de windows), en TRM borrar el API del TPS, Refrescar, bajar el TPS y reinstalar todo de 0 y limpio en el sistema.

    1. Failed to Execute Script pyi_rth_win32comgenpy: Este error está provocado debido a una librería faltante en el Windows instalado “pywin32”, para resolverlo se debe actualizar el Windows o al menos las librerías correspondientes a pythoncom o win32com . Este proceso es muy lento y demorado, una opción es utilizar un 2do PC como servidor y ya no el que está dando el problema, e incluso en algunas ocasiones es necesario formatear el PC si es que aún instalando las librerías no funciona.

    CSAT (Customer Satisfaction Score): Indicador de satisfacción inmediata que mide la experiencia del cliente tras una interacción o soporte recibido.

    Altas: Etapa posterior al Cierre, que marca el momento en que un nuevo cliente finaliza su onboarding y comienza a operar activamente con Toteat

  • Implementación

    IP: Internet Protocol. Dirección numérica única que identifica un dispositivo dentro de una red. En Toteat, cada impresora, PC o equipo conectado (como un POS o Totem) debe tener una IP fija o reservada para garantizar la comunicación estable entre el sistema, el servidor local y los periféricos.
    Firewall: Sistema de seguridad que controla y filtra el tráfico de red entrante y saliente según reglas definidas.
    Puertos: Canales lógicos que permiten la comunicación entre dispositivos y aplicaciones a través de una red. Cada servicio o proceso utiliza un número de puerto específico.
    Archivo host: Archivo del sistema operativo que asocia manualmente nombres de dominio con direcciones IP antes de consultar servidores DNS.
    DNS: Domain Name System, Sistema que traduce los nombres de dominio (como toteat.com) en direcciones IP numéricas que los equipos pueden entender.

    DHCP: Protocolo que asigna automáticamente direcciones IP a los dispositivos conectados; en Toteat se recomienda IP fija para impresoras y servidores.

  • Tech
    • API: Una API (Interfaz de Programación de Aplicaciones) es un conjunto de reglas y protocolos que permiten que diferentes aplicaciones de software se comuniquen entre sí. Las APIs definen cómo las solicitudes y respuestas deben ser formateadas, qué operaciones están disponibles, y cómo los datos deben ser intercambiados. Esto permite a los desarrolladores integrar funciones y servicios de otros programas sin tener que entender su código interno. Por ejemplo, una API de un servicio de mapas permite a una aplicación obtener información sobre ubicaciones y direcciones sin tener que construir un sistema de mapas desde cero. Las APIs son fundamentales en el desarrollo de software moderno, facilitando la creación de aplicaciones más complejas y conectadas.
    • Bases de datos: Las bases de datos relacionales y no relacionales son dos tipos de sistemas de gestión de bases de datos con diferentes enfoques para almacenar y gestionar datos. Las bases de datos relacionales, como MySQL y PostgreSQL, organizan los datos en tablas con filas y columnas, utilizando un esquema estructurado y soportando relaciones entre tablas mediante claves primarias y foráneas. Son ideales para datos con estructuras bien definidas y relaciones complejas. En contraste, las bases de datos no relacionales, como MongoDB y Cassandra, almacenan datos en formatos más flexibles, como documentos JSON, grafos o tablas de claves-valor. Son adecuadas para manejar grandes volúmenes de datos no estructurados o semi-estructurados y permiten un escalamiento horizontal más sencillo. La elección entre relacional y no relacional depende de las necesidades específicas del proyecto, como la naturaleza de los datos y los requisitos de escalabilidad y flexibilidad.
    • Caso de Prueba: Es un conjunto de acciones que se ejecutan para verificar una característica o funcionalidad particular de una aplicación de software.
    • Defecto (Bug): Error, fallo o defecto en una aplicación que hace que se comporte de manera inesperada. Los defectos tienen diferentes severidades (Impacto que tiene un defecto en el funcionamiento del sistema): Crítica - Alta - Media - Baja.
    • Escalamiento: El escalamiento horizontal y el escalamiento vertical son dos enfoques para aumentar la capacidad de un sistema. El escalamiento vertical, también conocido como "scaling up," implica aumentar la capacidad de un solo servidor añadiendo más recursos, como CPU, RAM o almacenamiento. Esto mejora el rendimiento del servidor individual, pero tiene un límite físico y puede ser costoso. Por otro lado, el escalamiento horizontal, o "scaling out," consiste en añadir más servidores al sistema para distribuir la carga de trabajo. Esto permite un crecimiento casi ilimitado y proporciona redundancia, mejorando la disponibilidad y la tolerancia a fallos, ya que si un servidor falla, los otros pueden seguir funcionando.
    • ETL: En particular ETL (Extract, Transform, Load) es el proceso con el cual estamos migrando los clientes desde legacy a new. Toma la data que esta en Datastore (la base de datos de legacy) y la traspasa a new a través de las mismas APIs que utiliza el TRM nuevo. Es un proceso relativamente caro en cómputo por la complejidad y cantidad de data que involucra (y la cantidad de llamados individuales a la API).
    • Flask: Flask es un microframework de Python diseñado para facilitar la creación de aplicaciones web de manera rápida y sencilla, sin imponer una estructura rígida o dependencias pesadas. Creado por Armin Ronacher, Flask es conocido por su simplicidad y flexibilidad, permitiendo a los desarrolladores construir desde pequeños proyectos hasta aplicaciones web complejas. Utiliza una arquitectura basada en WSGI y viene con un servidor de desarrollo incorporado, soporte para pruebas unitarias y una estructura modular que permite extender sus capacidades mediante extensiones. Flask se enfoca en proporcionar las herramientas esenciales necesarias para el desarrollo web, como el enrutamiento de URLs, la gestión de solicitudes y respuestas HTTP, y la integración de plantillas Jinja2, ofreciendo así una experiencia de desarrollo minimalista y potente.
    • Framework: Un framework de programación es una plataforma de software que proporciona una estructura y un conjunto de herramientas predefinidas para facilitar el desarrollo de aplicaciones. Actúa como una base sobre la cual los desarrolladores pueden construir programas, ahorrando tiempo y esfuerzo al reutilizar código común y seguir una arquitectura coherente. Los frameworks incluyen bibliotecas, componentes, y directrices que ayudan a resolver problemas comunes en el desarrollo de software, como la gestión de la base de datos, el enrutamiento de URL, y la interacción con la interfaz de usuario. Ejemplos populares de frameworks incluyen Django para Python, Spring para Java, y Angular para JavaScript. Estos frameworks permiten a los desarrolladores enfocarse en la lógica específica de sus aplicaciones, mejorando la productividad y la calidad del código.
    • Google App Engine: Google App Engine es una plataforma como servicio (PaaS) proporcionada por Google Cloud que permite a los desarrolladores construir y desplegar aplicaciones web y móviles sin preocuparse por la infraestructura subyacente. App Engine gestiona automáticamente el escalado, equilibrado de carga y mantenimiento de servidores, permitiendo a los desarrolladores centrarse en el código y la lógica de la aplicación. Soporta varios lenguajes de programación, como Python, Java, Node.js y Go, y proporciona servicios integrados como bases de datos, autenticación y almacenamiento. Con App Engine, las aplicaciones pueden escalar automáticamente para manejar grandes volúmenes de tráfico, asegurando alto rendimiento y disponibilidad sin intervención manual.
    • Google Cloud Datastore: Google Cloud Datastore es un servicio de base de datos NoSQL completamente gestionado por Google Cloud, diseñado para manejar grandes volúmenes de datos estructurados y no estructurados con alta disponibilidad y escalabilidad. Está optimizado para aplicaciones que requieren consultas complejas, transacciones ACID, y estructuras de datos jerárquicas. Cloud Datastore permite a los desarrolladores almacenar y consultar datos en tiempo real mediante una interfaz sencilla y consistente, sin preocuparse por la administración de servidores o el escalado. Se integra perfectamente con otros servicios de Google Cloud, como App Engine y Cloud Functions, proporcionando una solución robusta y flexible para aplicaciones web y móviles que necesitan manejar datos dinámicos y distribuidos.
    • Google Cloud Run: Google Cloud Run es un servicio de Google Cloud que permite ejecutar aplicaciones en contenedores de forma completamente gestionada, sin necesidad de administrar la infraestructura subyacente. Con Cloud Run, puedes desplegar aplicaciones escritas en cualquier lenguaje de programación, empaquetadas en contenedores, y ejecutarlas en una infraestructura escalable que ajusta automáticamente los recursos según la demanda. Cloud Run soporta aplicaciones sin estado, ideal para microservicios y APIs, y ofrece facturación basada en el uso, lo que significa que solo pagas por el tiempo que tu aplicación está en ejecución. Además, se integra fácilmente con otros servicios de Google Cloud, proporcionando una solución flexible y eficiente para el desarrollo y despliegue de aplicaciones modernas.
    • HTTP: HTTP (Hypertext Transfer Protocol) es un protocolo de comunicación utilizado en la World Wide Web para la transferencia de datos entre clientes (generalmente navegadores web) y servidores. Es el fundamento sobre el cual se basa la navegación web, permitiendo la solicitud y entrega de recursos como páginas HTML, imágenes, videos y otros contenidos. HTTP funciona mediante un modelo de solicitud-respuesta, donde el cliente envía una solicitud HTTP a un servidor, y este responde con el recurso solicitado o un mensaje de error si el recurso no está disponible.
    • JavaScript: Javascript es un lenguaje de programación de alto nivel y dinámico, fundamental para el desarrollo web. Creado por Brendan Eich en 1995, se ejecuta principalmente en los navegadores web y permite crear páginas interactivas y dinámicas. A diferencia de HTML y CSS, que se utilizan para estructurar y diseñar páginas web, JavaScript agrega funcionalidad a las páginas, permitiendo la manipulación del contenido, la validación de formularios, la creación de animaciones y la comunicación asincrónica con servidores mediante tecnologías como AJAX. Además, con la llegada de entornos de ejecución como Node.js, JavaScript también se utiliza en el desarrollo del lado del servidor, ampliando su uso más allá del navegador y permitiendo construir aplicaciones web completas.
    • JSON: JSON (JavaScript Object Notation) es un formato de intercambio de datos ligero y de fácil lectura, tanto para humanos como para máquinas. Utilizado principalmente para transmitir datos entre un servidor y una aplicación web, JSON se basa en una estructura de pares clave-valor y listas ordenadas, lo que permite representar datos complejos de manera sencilla y estructurada. Su sintaxis es derivada de JavaScript, pero es independiente del lenguaje de programación, lo que lo hace compatible con muchos lenguajes, como Python, Java y C#. Gracias a su simplicidad y eficiencia, JSON se ha convertido en el formato preferido para APIs y servicios web, facilitando la comunicación y el intercambio de datos en aplicaciones modernas.
    • La nube: La nube es un conjunto de servidores en Internet que proporcionan almacenamiento y servicios de computación, permitiendo a usuarios y empresas acceder a datos y aplicaciones desde cualquier dispositivo con conexión a Internet. En lugar de depender de hardware local, los datos y aplicaciones se alojan en servidores remotos gestionados por proveedores de servicios en la nube, como Amazon Web Services, Google Cloud o Microsoft Azure. Esto ofrece flexibilidad, escalabilidad y seguridad, ya que los recursos se pueden ajustar según las necesidades y los datos están protegidos con medidas avanzadas de seguridad y copias de seguridad.
    • Lenguaje de programación: Un lenguaje de programación es un sistema formal de comunicación diseñado para expresar instrucciones que pueden ser entendidas y ejecutadas por una computadora. Permite a los desarrolladores escribir programas que especifican las operaciones que la máquina debe realizar, utilizando una sintaxis y una gramática específicas. Estos lenguajes pueden ser de bajo nivel, cercanos al código máquina y específicos de una arquitectura, o de alto nivel, más abstractos y fáciles de entender para los humanos.
    • MongoDB: MongoDB es una base de datos NoSQL orientada a documentos que almacena datos en formato JSON flexible, permitiendo la gestión de datos semi-estructurados y no estructurados de manera eficiente. Es ideal para aplicaciones que requieren alta disponibilidad, escalabilidad horizontal y capacidad para manejar grandes volúmenes de datos con estructura dinámica. MongoDB Atlas es el servicio de base de datos en la nube gestionado por MongoDB, que facilita la implementación, operación y escalado de bases de datos MongoDB en plataformas de nube como AWS, Azure y Google Cloud. Atlas automatiza tareas administrativas como la configuración, las actualizaciones, el monitoreo y las copias de seguridad, permitiendo a los desarrolladores concentrarse en construir sus aplicaciones mientras disfrutan de una infraestructura de base de datos segura y de alto rendimiento.
    • Paradigma cliente-servidor: El paradigma cliente-servidor es un modelo de comunicación en redes donde un dispositivo, llamado cliente, solicita servicios o recursos de otro dispositivo, llamado servidor. El servidor, que almacena datos y aplicaciones, espera solicitudes de uno o más clientes y responde proporcionándoles los recursos solicitados. Este modelo es fundamental en muchas aplicaciones de Internet, como sitios web y correos electrónicos, donde los clientes (navegadores web o aplicaciones de correo) se comunican con servidores para acceder a contenido, enviar mensajes o realizar transacciones, distribuyendo así la carga de trabajo y optimizando el uso de recursos en la red.
    • Plan de pruebas: Conjunto de casos de prueba a realizar en una funcionalidad o sistema determinado para comprobar su correcto funcionamiento.
    • Pruebas Funcionales: Son pruebas específicas, concretas y exhaustivas para probar y validar que el software hace lo que debe y sobre todo, lo que se ha especificado.
    • Pruebas de Regresión: Pruebas realizadas para confirmar que los cambios recientes en el código no han afectado funcionalidades existentes.
    • Pruebas de Integración: Este tipo de pruebas evalúa la forma en que interactúan y operan varios módulos de aplicaciones de software de forma cohesiva. Un ejemplo de estas prueba sería probar órdenes de Rappi y ver como ingresan a TRM.
    • Pruebas Automatizadas: Pruebas creadas con una herramientas específica, en nuestro caso utilizamos Cypress, para luego ser ejecutadas automáticamente, sin intervención humana.
    • Python: Python es un lenguaje de programación de alto nivel, interpretado y de propósito general, conocido por su simplicidad y legibilidad. Creado por Guido van Rossum y lanzado por primera vez en 1991, Python utiliza una sintaxis clara y concisa que facilita la escritura y el mantenimiento del código, haciéndolo ideal tanto para principiantes como para desarrolladores experimentados. Es muy versátil y se usa en una amplia variedad de aplicaciones, desde desarrollo web y análisis de datos hasta inteligencia artificial y automatización de tareas. Además, cuenta con una extensa biblioteca estándar y una activa comunidad de desarrolladores que contribuyen con numerosos paquetes y herramientas para ampliar sus capacidades.
    • Quality Assurance (QA): Es una disciplina clave en el desarrollo de software y productos que se centra en asegurar que los productos cumplen con los requisitos y estándares de calidad establecidos. El objetivo principal de QA es prevenir defectos y problemas antes de que el producto llegue al cliente final.
    • RFP (Request for Proposal): es un documento interno que tiene por objetivo solicitar el desarrollo de un nuevo requerimiento/ funcionalidad o la resolución de un bug. Este documento debe proporcionar toda la información necesaria para que los equipos de 🤖Engineering (toteat) y Producto puedan evaluar y analizar el pedido realizado.     • Vue: Vue.js es un framework progresivo de JavaScript para construir interfaces de usuario y aplicaciones de una sola página (SPA). Creado por Evan You, Vue.js es conocido por su enfoque flexible y su facilidad de integración, ya que puede ser utilizado tanto para mejorar aplicaciones existentes como para desarrollar proyectos completos desde cero. Vue.js utiliza una sintaxis declarativa similar a HTML para vincular datos y manejar eventos, y ofrece características avanzadas como componentes reutilizables, reactividad de datos y enrutamiento para gestionar la navegación en las aplicaciones. Su arquitectura basada en componentes facilita el desarrollo de interfaces de usuario complejas y la gestión del estado de la aplicación, mientras que su ecosistema de herramientas y bibliotecas, como Vue Router y Vuex, proporciona soluciones robustas para la gestión del enrutamiento y el estado. La simplicidad y eficiencia de Vue.js lo han convertido en una opción popular entre los desarrolladores web.
    • Webapp2: Webapp2 es un framework ligero de Python diseñado para el desarrollo de aplicaciones web, conocido por su simplicidad y flexibilidad. Originalmente creado para Google App Engine, Webapp2 proporciona una estructura minimalista que facilita el manejo de solicitudes HTTP, el enrutamiento de URLs y la gestión de sesiones. Su diseño modular permite a los desarrolladores añadir solo las funcionalidades que necesitan, lo que lo hace adecuado para aplicaciones que requieren una huella pequeña y alto rendimiento. Además, Webapp2 es compatible con WSGI, el estándar para servidores web en Python, lo que permite su uso en una variedad de entornos de despliegue. Aunque es menos conocido que frameworks más grandes como Django o Flask, Webapp2 es valorado por su eficiencia y facilidad de uso en proyectos de tamaño pequeño a mediano.

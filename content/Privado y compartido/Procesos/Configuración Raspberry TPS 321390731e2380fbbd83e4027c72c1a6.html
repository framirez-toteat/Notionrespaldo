  • Configuración Raspberry

    Recomendación: Conectar teclado y mouse a la raspberry para facilitar configuración

    1. Abrir “Terminal” en la parte superior izquierda e ingresar al usuario administrador root con el comando:


    Sudo Su -

    1. Estando ya en el administrador (root), descargar el paquete de la VPN que utilizamos para este formato, Tailscale:

    curl https://tailscale.com/install.sh | sh

    Se comenzará a descargar el paquete, finalizando con un “Instalación completa”

    Si falla en algún punto, validar la conexión a internet del equipo.

    1. Una vez esté Instalado, se debe configurar la key de este ambiente en el VPN, cada ambiente tiene una Key única, actualmente se otorga esta key manualmente previa solicitud a Carlos Zager (12/03/26), y se debe colocar el siguiente comando:

      tailscale up --login-server https://rs1.toteat.com:7654 --authkey NUMERODEKEY

    Si la key está mal ingresada, enviará un error de Authkey not found

    Si la key está ok, avanzará a la sgte línea.

    1. Aquí comenzará a descargar todas las configuraciones asociadas al TPS, con haber avanzado a la sgte línea (foto anterior), sabemos que está descargando, visualmente no se ve una barra de descarga o información del estado actual. Para saber que la descarga terminó correctamente, se debe esperar a que en el escritorio se descarge un archivo llamado “TPS.txt”, el cual confirma que está ya completado el proceso de descarga.

    1. Cuando esté ya el archivo en el escritorio, se debe dar permisos al usuario que administrará las impresoras con:

      sudo usermod -aG lpadmin toteat
  • Configuración Portainer e impresoras

    Importante: Para las siguientes configuraciones se debe estar dentro de la VPN

    1. Obtener IP de la raspberry en la VPN, para eso accederemos desde nuestro equipo conectado a la VPN a http://100.64.0.1:10001/nodes, aquí, nos aparecerán todas las máquinas dentro de la VPN por lo que usando el buscador del navegador, conseguir el ID del local que estamos instalando (recordar que cada ID tiene su key, por lo que solo debiera aparecer uno con ese nombre/ID).

    Este cuadro muestra:
    a. ID local
    b. IP dentro de la VPN de esta raspberry (es única y no reciclable)
    c. Si está conectada o no a la red
    d. Soporte de la raspberry
    e. Reinicio de TPS y de Cups

    Revisar: Soporte a las raspberrys

    1. Obteniendo la IP de la raspberry ingresaremos http://100.64.0.1:9000 donde crearemos el acceso en portainer a la raspberry correspondiente, aquí pondremos nuestro usuario y contraseña, y accederemos a Environments

    Y aquí rellenaremos el docker con los datos obtenidos desde “nodes”, importante, a la IP se le debe asignar el puerto 9001, como se ve en la siguiente imagen:

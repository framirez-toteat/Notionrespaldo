1. Descripción

Listado de validaciones que puede realizar el personal del local, antes de escalar la solicitud, validaciones básicas y simples que puedan corregir errores eventuales sin un asesoramiento por parte del Staff Toteat.

2. Alcance

Debe aplicarse en cada incidencia que tenga el cliente, con tal de asegurarse que todo lo básico de configuración y conexión esté correctamente realizado.

3. Entrada

Este proceso debe ser realizado al identificar un error en la operatividad del sistema, escalado por el usuario de Toteat en el restaurante, ya sea Cajero, mesero, etc.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Determinar correctamente cual es el PC Servidor.
    • Determinar correctamente cual es la red Wifi/LAN a la que corresponde el PC Servidor.
    • Verificar visualmente el estado de los equipos, que estén encendidos y conectados.
    • Tener usuario/contraseña de administrador en caso de necesidad.

5. Procedimiento Básicos

  • Paso 1: Validaciones operativas
    • Verificar el que el usuario sea el correcto.
    • Verificar que esté en el ambiente correcto.
    • Verificar que el usuario tenga turno abierto.
    • Verificar que el usuario tenga caja abierta.
  • Paso 2: Validaciones de configuración
    • Verificar si el PC principal está encendido.
    • Verificar el que esté en la red Wifi correcta.
    • Verificar que esté en la URL correcta (res3.toteat.com, res4.toteat.com, etc)

6. Procedimiento Avanzados

  • Paso 1: Revisión TPS (Toteat Print Server)
    • Verificar que TPS esté ejecutado correctamente. Si está ejecutado, reiniciarlo:
    • Si no está ejecutándose, abrir el archivo en el escritorio:
    • Con el software ejecutado debiesen corregirse problemas de impresión y estados Online/Offline, si aún no se resuelven los problemas o no se ejecutó correctamente el software ingresar al chat de soporte.
  • Paso 2: Impresora USB en error de conexión

    • Verificar visualmente impresora, cables conectados y luces de actividad encendidas, verificar intermitencia de luz de error. Verificar papel, tapa de papel, y cualquier impedimento físico que pueda tener.
    • Verificar el puerto al que está la impresora conectado, validar en su configuración en Windows si corresponde al USB correcto. Cambiar de puerto si es necesario.
    • Punto de control: Imprimir página de prueba de Windows en la impresora, si esta no sale sigue siendo error de conexión, verificar/cambiar cable, puerto, conectores, etc. Si la página de prueba de Windows sale, debiese estar corregido el error.
    • Dispositivo USB no reconocido: Al mostrarse esta alerta en Windows, se debe proceder a reinstalar los drivers del puerto USB.
      • Paso 1: Ir al Administrador de tareas buscar el puerto USB con problemas y reinstalarlo
      • Paso 2: Si el problema persiste se debe indicar a cliente que cambie de puerto USB la impresora, si a pesar de esto no funciona se debe cambiar el cable USB por uno en mejor estado
  • Paso 3: Impresora Ethernet en error de conexión
    • Verificar visualmente impresora, cables conectados y luces de actividad encendidas, verificar intermitencia de luz de error. Verificar papel, tapa de papel, y cualquier impedimento físico que pueda tener.
    • Verificar que IP de la impresora corresponda con la IP configurada en Windows para ella:
    1. Obtener Selftest de la impresora:

      1.1. Apagar impresora.

      1.2. Apretar botón feed.

      1.3. Encender impresora manteniendo botón feed pulsado.

      1.4. En cuanto salga el Selftest, dejar de presionar el botón feed.

      1.5. Presionar el botón feed nuevamente un segundo, dejando salir papel en blanco y asegurándonos que no esté trabada.

    1. Verificar IP en Windows en Propiedades de Impresora:

      2.1. Ir al Panel de dispositivos de impresoras (Tecla win + “R” Comando: Control printers)

      2.2. Click derecho en la impresora correspondiente

      2.3. Propiedades de la impresora

      2.4. Pestaña “Puertos”, y aquí verificar si la IP es la misma que en el selftest.

      2.5 Si la IP no coincide se debe hacer coincidir cambiándole la IP a la impresora, o desde la configuración de Windows.

      2.6 Si la IP si coincide se debe validar que estén dentro de la misma red y validaciones físicas de conexión, cables, puertos, conectores, etc.

    • Punto de control: Imprimir página de prueba de Windows en la impresora, si esta no sale sigue siendo error de conexión, verificar/cambiar cable, puerto, conectores, etc. Si la página de prueba de Windows sale, debiese estar corregido el error.
    • Colocar la IP de la impresora en el comentario de la impresora si es que hubo cambios.
  • Paso 4: Revisión de que no hayan cambiado los nombres en el sistema de las Impresoras.

    A veces al desconectar y conectar una impresora el sistema operativo Windows la renombra y queda con un nombre distinto al configurado en Toteat. (Menú lateral > Configuración > Impresoras). Revisar caracteres especiales en la impresora de Windows símbolos como *, Tildes, + , etc.

  • Paso 5: En caso de impresoras térmicas.

    Revisar que el papel esté correctamente insertado, con el lado térmico al frente. Puede comprobarlo rayando con la “uña”; si cambia de color el lado frontal está bien colocado. Si no cambia de color en ningún lado, el papel no es térmico, debe cambiarlo por el correcto.

  • Paso 6: Verificación de Toteat Access Server (opcional)

    Si se usa un sistema de login rápido con códigos cortos o tarjetas NFC debe verificar que la aplicación ToteatAccessServer.exe esté ejecutándose.

7. Salida

  • Se valida correctamente lo básico y avanzado previo a una revisión por parte del área SAC, con la cual ya tiene nociones de lo realizado y así minimizar el rango de búsqueda del problema.

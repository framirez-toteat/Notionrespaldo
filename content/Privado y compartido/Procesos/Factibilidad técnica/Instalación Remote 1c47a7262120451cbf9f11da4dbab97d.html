1. Descripción

En la primera parte de revisión de factibilidad técnica del cliente, se deben realizar dos monitoreos para así validar que este cumpla con parte de los requisitos mínimos que Toteat necesita para trabajar correctamente, estos monitoreos son:

  1. Monitoreo de red: Este consta de monitorear la estabilidad de la señal que tiene el cliente en el equipo principal, con el fin de determinar qué tan estable es la red y/o cuanta intermitencia tiene dentro de un rango de tiempo en específico, el monitoreo se aprueba si es que la red está conectada de corrido sin intermitencias por aproximadamente 8 horas, si no, el cliente debe hacer las modificaciones correspondientes para acercarse a ese rango (cablear la el PC al router, despejar el router, validar con el proveedor, etc.)
  1. Monitoreo del equipo Principal: La herramienta utilizada para realizar el monitoreo de red también nos proporciona toda la información de hardware que el equipo en el que está instalado tiene, para así, determinar si el equipo cumple con los requisitos mínimos para ser utilizado como “servidor” o no, de ser así deberá cambiar el equipo por uno que si se acerque más a estos. Validar con 📗Requisitos mínimos recomendados .

Para realizar este monitoreo es que se usará una herramienta externa llamada Screenconnect (o Remote por su nombre anterior) la cual además de servir para validar estos monitoreos, es la herramienta que el equipo de SAC utiliza para hacer el soporte remoto a los clientes en caso de que sea necesario.

2. Alcance

Este proceso se debe aplicar para todos los clientes en la primera instancia de conversaciones con el ejecutivo de ventas, de este monitoreo pueden alertarse inconvenientes que podrían afectar la experiencia de usuario en el uso de Toteat.

3. Entrada

Este proceso parte a través del envío de un correo por parte del ejecutivo de ventas el cual trae un código para la instalación de Remote, y un pequeño explicativo del como realizarlo.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Se debe realizar idealmente en el PC que será el servidor en el uso de Toteat
    • Se debe realizar en la red definitiva que tendrá el cliente en su local (por ej: no lo debe realizar en su casa si es que el local físicamente está en otro lugar)
    • Se debe realizar idealmente en un PC con Windows, teniendo en consideración que es un requisito mínimo para ser servido

5. Procedimiento

  • Paso 1: Recopilación información
    • Desde el correo que envía el agente de ventas, se debe extraer la información asociada a:
    1. El código de Remote: Es un código de 5 dígitos único por cliente que será utilizado para obtener el archivo de descarga de la herramienta.
    1. Link de descarga: La URL necesaria para realizar la acción de descarga del archivo de Remote, para todos los clientes la dirección es la misma Toteat.screenconnect.com
    • El cuerpo del correo que se envía actualmente es:

    Instrucciones:

    1. Haz click aquí para ir a la web de conexión remota 
    2. Introduzca el siguiente código de acceso:  44595 (código de prueba para el ejemplo)
    3. Una vez ingresado el código pulse sobre la flecha azul
    4. Se iniciará la descarga del software bajo el nombre ToteatRemote.Client.exe
    5. Luego de realizada la descarga, ejecute el instalador para continuar. Si aparece un mensaje solicitando permiso para realizar cambios en el equipo, seleccione "Sí"
    6. Una vez instalado Toteat Remote aparecerá el ícono de la aplicación en la bandeja de sistema del computador, y una notificación indicando que se ha realizado la conexión.

  • Paso 2: Ingreso al URL y descarga de archivo
    • Ya con el código, se ingresará a Toteat.screenconnect.com y se ingresará el código en el cuadro de texto “Unirse con un código”.

    • Al ingresar el código la flecha al costado derecho del cuadro de texto pasara de gris a rojo, y clickeando sobre ese ícono comenzará la descarga.
  • Paso 3: Ejecutar el archivo descargado
    • Una vez la descarga esté lista, se debe clickear sobre el ícono de ella para así comenzar la instalación.

    • Al aparecer la ventana de permisos debe clickear sobre el icono de ‘Sí’ .
    • Con esto se instalará el archivo correctamente, para validar que esté ok, se debe verificar que el ícono del archivo se esté ejecutando correctamente en los íconos de la parte inferior derecha .

    • Nota: “Este proceso puede tardar más o menos dependiendo de la velocidad de descarga de internet y del procesador del equipo, en general no debiese tardar más de un par de minutos.”

6. Salida

  • Se logra instalar el remote de manera correcta para iniciar el monitoreo.

7. Excepciones del flujo

  • Clientes que no requieran PC servidor (por ejemplo, usarán solo un equipo IMIN).
  • Clientes específicos dictaminados por el country manager identificados como excepción en Salesforce.

8. Puntos de control

  • Para validar el proceso se debe verificar el ícono de Remote abajo a la derecha.
  • Se puede consultar el estado del remote a un agente SAC para validar si existe conexión a dicho código.

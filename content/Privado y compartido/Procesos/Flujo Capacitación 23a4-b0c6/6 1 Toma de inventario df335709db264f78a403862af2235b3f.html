1. Descripción

Este proceso tiene como objetivo garantizar la actualización precisa del inventario dentro del sistema. Se llevará a cabo mediante la realización periódica de tomas de inventario, en las cuales se deberá registrar las cantidades físicas de los insumos o ingredientes disponibles. Esto permite mantener un control eficiente y actualizado de los niveles de stock, asegurando que la información en el sistema refleje fielmente la realidad.

2. Alcance

Este proceso abarca todas las actividades relacionadas con la actualización y control del inventario físico dentro del sistema. Involucra la identificación y conteo de insumos o ingredientes disponibles en las bodegas, así como el registro de dichas cantidades en el sistema. El proceso se aplica de manera periódica y está destinado a garantizar que los datos de inventario reflejen con precisión el stock físico, permitiendo una correcta gestión de recursos y minimizando errores en la planificación de compras y producción.

3. Entrada

Las entradas necesarias para ejecutar este proceso incluyen:

  • Listados de inventario previos: Información sobre los insumos o ingredientes existentes en las bodegas, que servirán de referencia para el conteo actual.
  • Personal asignado: Personas responsables de realizar el conteo físico de los insumos o ingredientes en las bodegas.
  • Acceso al sistema de inventario: Permisos y acceso al sistema para registrar las cantidades actualizadas, realizar ajustes y aprobaciones según corresponda.

Estas entradas garantizan que el proceso pueda ejecutarse de manera precisa y eficiente.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Identificación y configuración de insumos en sus bodegas.
    • Permisos para poder realizar tomas de inventario y aprobaciones.
    • Configuración de unidades de medida para la toma.

5. Procedimiento

  • Paso 1: Toma de inventario
    • Se deberá ir a la ruta Stock Control – Inventarios.
    • Allí estará la opción Toma de inventarios.
    • En está sección, podremos seleccionar la fecha en la cual se cargara la toma, asi como a que bodega se realizara y formato de ingreso masivo.
    • Se podra ingresar la opción de mostrar minimo y maximos por Bodega o Total y rellenar vacios con 0.
    • Nota: Se debe tener en cuenta que, al seleccionar la fecha de toma y cargar este inventario, será en esa fecha donde se cargará el inventario al comienzo del día. Por lo tanto, si se carga una toma de inventario hoy en la tarde, se descontará todo lo que se vendió en la mañana.
    • Nota: Si no se tiene stock de un insumo, no se puede dejar el espacio en blanco, por lo que se debe rellenar con un 0, de ser necesario.

  • Paso 2.1: Ingreso de información Masiva
    • Lo primero, seleccionar la fecha y la bodega.
    • Se debe dar click en el boton “XLS” y se descargara un archivo excel.
    • Al abrirlo, se debe rellenar la columna “Cantidad”.
    • Se pueden dejar campos en blanco para luego presionar la opción "Rellenar vacíos con 0" o hacerlo desde el mismo Excel.
    • Se copian todos los campos, sin encabezado, para luego ir al sistema y presionar "Subir Toma".
    • Se aparecerá la vista anterior de la imagen, para dar clic en "Grabar Toma".
    • Una vez presionado el botón, el sistema redirigirá a "Históricos".
    • Nota: Si se necesita cambiar la unidad de medida de un insumo al realizar la toma, se debe hacer en la configuración interna del insumo o ingrediente (revisar proceso 5.3).
    • Nota: Si es necesario, se pueden utilizar decimales al ingresar el stock.

  • Paso 2.2: Ingreso de información desde el sistema
    • Primero, seleccionar fecha y bodega.
    • Estando en Toma de inventario, se debe rellenar las cantidades que tenemos en nuestro stock.
    • Una vez listo, se presiona “Grabar Toma”.
    • Nota: Si se utiliza el filtro de búsqueda del sistema, se debe eliminar lo que se haya escrito antes de grabar, ya que, de otra manera, no permitirá grabar la toma.

  • Paso 3: Históricos
    • En Históricos, se podrán visualizar todos los movimientos relacionados con tomas de inventario, transferencias entre bodegas, transferencias entre locales y transformaciones de productos (declaración de producción).

    Cada uno de estos movimientos mostrará:

    • ID Movimiento: ID único proporcionado por el sistema para cada movimiento que se registre en Históricos.
    • Clase: Tipo de movimiento que se registro en Históricos.
    • Fecha: Fecha en la cual el movimiento afectará el inventario.
    • Bodega: Bodega o bodegas afectadas con el movimiento.
    • Hora Creación: Hora y fecha de creación del movimiento.
    • Usuario Creador: Usuario que ingresó y generó el movimiento.
    • Hora de Aprobación: Fecha y hora correspondientes a cuando se aprobó este movimiento.
    • Usuario Aprobador: Usuario que realizó la aprobación del movimiento.
    • Nota: Si alguno de estos movimientos no cuenta con los campos de “Hora Aprobación” y “Usuario Aprobador”, no se vera afectado el inventario, por lo que es como que nunca se hubiera realizado.
    • Nota: Si el nombre del usuario en el movimiento se visualiza en números, es porque ese usuario no es parte del staff, por lo que puede ser alguien de Toteat o un usuario eliminado.

  • Paso 4: Lápiz del movimiento
    • Al hacer clic en el ícono de lápiz, se ingresará al movimiento, donde estará la opción de revisar las cantidades ingresadas, modificar dichas cantidades y la fecha, si es necesario.
    • Al desplazarse hacia abajo, se podrá ingresar un comentario al movimiento, además de visualizar una serie de botones.
    • Confirmar/Autorizar Transacción: Se está autorizando este movimiento, por lo tanto, se generará una fecha de aprobación y un usuario aprobador. En consecuencia, el movimiento afectará los inventarios.
    • Grabar Cambios: Se graban las modificaciones realizadas, pero no se aprueba. Es decir, se guarda y se puede continuar editando más tarde.
    • Eliminar: Se elimina el movimiento.
    • Cancelar: No se guardan cambios en el movimiento.

  • Paso 5: Aprobado
    • Una vez el movimiento haya sido autorizado, se visualizara en Históricos asi:
    • Si se ingresa al ícono de lápiz de este movimiento, se observará que ya no es posible realizar modificaciones de ningún tipo.
    • Al bajar solo veremos dos botones “Reversar Confirmación” y “Ok”.
    • Reversar Confirmación:Se revierte la autorización del movimiento, por lo que se puede modificar nuevamente.
    • Ok: Simplemente devuelve a Históricos.
    • Nota: Al revertir un movimiento, se pueden modificar las cantidades y la fecha. Esto se utiliza cuando faltó ingresar cantidades de insumos o se necesita realizar un ajuste.
    • Nota: Es necesario tener un perfil de dueño o creador para poder eliminar o desautorizar un movimiento.

6. Salida

  • Stock actualizado según la información física del cliente.
  • Valorización de bodegas.

7. Excepciones del flujo

  • No hay excepciones.

8. Puntos de control

  • Revisar en Históricos que el movimiento fue aprobado correctamente.

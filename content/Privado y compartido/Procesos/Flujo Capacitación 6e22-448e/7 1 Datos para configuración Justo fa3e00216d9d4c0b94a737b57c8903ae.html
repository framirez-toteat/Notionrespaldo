1. Descripción

En este módulo, abordaremos los campos de configuración más importantes durante la activación de la integración de Toteat- Justo.

2. Alcance

Para iniciar el proceso de integración con Justo, es crucial comprender las configuraciones necesarias que garantizarán un proceso exitoso. Las configuraciones de las APIs incluyen campos estándares que siempre deben configurarse, así como campos específicos que requieren datos exclusivos de Justo al igual que el cliente requiere datos de nuestra API para configurarlos en Justo.

3. Entrada

Cuando el cliente desea iniciar su operación con la integración de Justo, comienza este proceso de solicitud y recopilación de datos para las configuraciones de Justo.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integraciones y tener conocimientos sólidos sobre el proceso de activación en la integración Justo-Toteat.
    • Confirmar que el cliente ya esté creado y activo en Justo.
    • Verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de iniciar la integración.
    • Si el cliente está asociado a otro POS, solicitar que realice la baja antes de proceder con nuestra integración.
    • Asegurar que el cliente tenga los datos necesarios para la integración: Api Key, Webhooks, menús, y precios por lista para Justo.
    • Crear la API de Justo en Toteat para generar la URL y el Token, los cuales son datos necesarios para la configuración de la integración con Justo.
    • Confirmar con el cliente que los productos tengan imágenes.
    • Tener a disposición los datos de Toteat y Justo.
    • Recomendar reuniones cortas vía meet para aclarar dudas y explicar la configuración y uso de los datos en la activación.
    • Programar la integración para inicios de semana e informar al cliente sobre el tiempo de activación de 1 a 3 días hábiles.

5. Procedimiento

  • Paso 1: Intercambio de Datos vía correo.
    • El agente encargado de realizar la activación de la integración Justo-Toteat debe crear la api en Toteat, Ver 📗2.2. ¿Cómo se configura un API?

      Nota: Recordar que las Apis de Justo inician con 14XX, ejemplo: 1401, 1402, etc.

    • Al crear la APi para Justo se generan 2 datos importantes que se deben indicar al cliente en el correo, los cuales son:
      URL y Token de Toteat (Llave secreta en Justo)
    • Del cliente se requiere la siguiente información:

    1.- Api Key de Justo (este se genera una sola vez, cuando el cliente lo crea debe copiarlo de una vez y guardarlo ya que es un dato que luego no es visible completo en Justo):
    Menu lateral del admin de Justo - - - Plataforma - - - Api - - - Api Keys - - - Crear.

    2.- Los datos de Id de Menú externo y Id externa restaurantes lo puede indicar el cliente o se puede consultar por versiones por medio del Api key.

    3.- Precios por lista a usar en Justo, para ello se debe crear el precio por lista en el ambiente e identificarlo con el nombre de la marca-Justo, posteriormente, se debe descargar en formato de Excel, el archivo de precio por lista y enviarlo al cliente.

    • Nota: El cliente debe responder a lo solicitado a través del mismo correo, con el fin de mantener un registro formal de los datos.

6. Salida

  • Disponer de los datos de configuración para la integración de Justo es fundamental para agilizar y garantizar una ejecución correcta. Esta información permite realizar el proceso rápidamente, asegurando que se cumplan los parámetros establecidos. Así, no sólo se optimiza el tiempo empleado en la activación de la integración, sino que también se asegura la precisión y fiabilidad del proceso.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de Justo.

8. Puntos de control

  • Validación con el cliente del proceso de obtención y ubicación de los datos de configuración para la integración Justo-Toteat.

1. Descripción

En este módulo abarcáremos los campos a configurar más importantes durante la configuración de una API.

2. Alcance

En las configuraciones de las APIs, existen campos que se deben configurar siempre, independientemente de si se trata de una API para Delivery o una API estándar.

3. Entrada

Personal de Toteat enfocado en el proceso de configuración de APIs y activación de integraciones.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe cumplir con los siguientes requisitos:
    • Pertenecer al equipo Toteat.
    • Tener conocimientos del software Toteat.
    • Personal autorizado para la activación de los procesos de integraciones.

5. Procedimiento

  • Paso 1: Fuente de Origen
    • La fuente de origen ayuda a definir la reportería y la operación.
    • Para la configuración de la fuente de Origen se deben llevar a cabo los siguiente pasos:
    1. Dar clic en Menú Principal  >>> Ajustes Generales >>> General.
    1. En la sección General, se encuentra un item llamado “Delivery”, esta opción se debe activar.
    1. Después de activar, aparece una sección llamada "Fuentes de Origen". Allí, validamos la Aplicación a Integrar en la lista desplegable y la añadimos haciendo clic en el botón "+". Si se trata de un ecommerce que no está por defecto, se puede crear de manera personalizada con un custom.

    Nota: Recuerda siempre guardar los cambios realizados.

  • Paso 2: Forma de pago
    • La forma de pago es exclusiva para cada integración.
    • Para configurar la forma de pago en el caso de las integraciones se maneja un método único identificado con el nombre del delivery, varios ya se encuentran predefinidos.
      Si se trata de un ecommerce que no está por defecto, se puede crear de manera personalizada con un custom y el código que se le asigna a la forma de pago se debe tener en cuenta para indicarlo al ecommerce y/o empresa que vaya a utilizar el Api Standard.

    Nota 1: Verificar que la forma de pago quede activa y en el caso de los deliverys que quede con la opción de “Solo uso en Integraciones” activa.

    Nota 2: Recuerda siempre guardar los cambios realizados.

  • Paso 3: Sectores
    • Los Sectores nos permiten segmentar los pedidos en curso en la pantalla de órdenes de Toteat y así dar un mejor manejo a la operación Delivery, ademas de poder visualizar de manera mas amigable las ventas en los reportes de Deliverys.
    • Se recomienda crear los sectores con una jerarquía principal de INTEGRACIONES, y crear sub Jerarquías por cada uno de las marcas y luego por cada Delivery, como se observa a continuación:
    • Nota: Recuerda siempre guardar los cambios realizados

  • Paso 4: Precios x lista
    • Los precios por lista son muy útiles para filtrar y obtener reportes de delivery más organizados, por lo que es importante identificarlos de la manera más detallada posible.
    • Los precios por lista se recomiendan crearlas por marca y por plataforma.
    • Los productos y descripciones se verán en las distintas plataformas con el mismo nombre que tienen asignado en Toteat y bajo las jerarquías que se determinen.
    • Se recomienda cumplir con las restricciones que determina cada plataforma para los nombres de productos, descripciones, extras, nombres y número de jerarquías.
      Las imágenes deben ser cargadas en Toteat y deben ser la que el cliente desea que sean visibles en las diferentes plataformas.

    Notas:

    • Al crear los precios por lista, se debe indicar el precio que se desea en la plataforma de delivery.
    • Si no deseas enviar un producto a la integración, deja el campo de precio por lista vacío. Si colocas 0, se enviará con precio cero.
    • Si un producto está sin stock en una de las marcas se debe marcar la opción "Bloqueado" en el precio por lista que corresponda. Si deseas que no esté disponible para ninguna de las marcas y en ninguna plataforma se debe inactivar desde el maestro de productos.

  • Paso 5: Configuración de Turnos
    • Para la Integración solo aplica en el caso de Ubereats.
    • Recordar que los turnos que manejan en Uber deben quedar como sub turnos y con la opción de Delivery activa ( D ) solamente.
    • Nota: Recuerda siempre guardar los cambios realizados.
  • Paso 6: Dark Kitchen.

    Para crear una dark kitchen se debe realizar desde la sección de la Toteat Shop en toteat con la ayuda de la consola de Google Chrome, sigue estos pasos detallados:

    1. Ve a la sección de Toteat Shop>>>Configuración detallada.
    1. Una vez allí, ingresa al menú de configuraciones de Google Chrome en la parte superior derecha de la pantalla.
    1. Selecciona "Más herramientas" y luego "Herramientas para desarrolladores".
    1. En la pestaña que se abre, haz clic en el botón "Clear Console" para limpiar la consola.
    1. Luego escribe el comando S.creaDarkKitchen() y presiona enter.
    1. Para modificar el nombre del restaurante en la pestaña superior, es necesario activar la aplicación Toteat Shop. Una vez allí, podrás visualizar el campo para indicar el nombre del restaurante o de la marca, luego se puede desactivar.

    Recuerda que estas pestañas se crean de acuerdo a la cantidad de marcas que se manejan en el ambiente.
    Se deja una pestaña principal que se recomienda tenga el nombre del ambiente y las siguientes pestañas serán las destinadas para los nombres de las dark kitchen, esto es debido a que las dark kitchen deben tomar id diferente a cero (0) .
    • Nota: Recuerda siempre guardar los cambios realizados. Este proceso es realizado por personal autorizado de Toteat.
    1. Con la configuración de la dark kitchen lista, en la API ya aparece el campo de “Dark Kitchen” y se puede seleccionar en la lista desplegable la marca que se desea configurar.
  • Paso 7: Creación de la API
    • Ingresar Menú Principal  >>> Configuración >>> Print Server y API.
    • Ingresar a la pestaña API Config.
    • Desplegar la lista de tipos de APIs y seleccionar la que se requiere:
    • Dar clic al botón “Agregar Api”.
    • En ese momento aparece la pantalla para configurar la Api, se observan 4 pestañas: General/Pedidos/Productos/Seguridad.
    • En la sección “General” se define el nombre de la Api, la cual se recomienda indicar la marca y la plataforma o Ecommerce, ej: Marca|Plataforma (1). También, la selección de la Dark Kitchen (2) configurada en el paso 6.
      Como datos adicionales para temas operativos, si bien no son obligatorios, pero son de gran ayuda, son:
      - El email para errores (3), por medio del cual se reporta algunos errores de las integración, y la URL servicio externo, este último seria el link de la plataforma.
    • Dependiendo de la plataforma a integrar hay datos más específicos que son otorgados por las plataformas al restaurante y deben indicarse para poder configurarlos en la Api (Id Externa, Tokens, Webhooks) (4).

    • En la sección de “Pedidos” se definen los campos de fuente de origen (1), forma de pago (2) y sector (3).
    • En la sección de “Productos” se define el campo de Lista de precios.

6. Salida

  • Una API configurada correctamente permite que el sistema Toteat se comunique e intercambie datos de manera eficiente esto facilita la integración de diferentes servicios y plataformas, lo que puede resultar en una mejor experiencia para los usuarios finales.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.

8. Puntos de control

  • Evaluación de homologación de conocimientos.
  • Capacitaciones.
  • Activaciones de integraciones.

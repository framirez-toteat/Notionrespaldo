6.1. Datos para configuración PeYa

PropietarioFFrancisco Ramirez
Etiquetas

1. Descripción

En este módulo, abordaremos los campos de configuración más importantes durante la activación de la integración de Toteat con Pedidos Ya.

2. Alcance

Para iniciar el proceso de integración con Pedidos Ya es crucial comprender las configuraciones necesarias para garantizar un proceso de integración exitoso.
Las configuraciones de las APIs incluyen campos estándar que siempre deben configurarse, así como campos específicos que requieren datos exclusivos de Pedidos Ya.

3. Entrada

Cuando el cliente desea iniciar su operación con la integración de Pedidos Ya, comienza este proceso de solicitud y recopilación de datos para las configuraciones para llevar a cabo la integración.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integraciones y tener conocimientos sólidos sobre el proceso de activación en la integración PeYA-Toteat.
    • Confirmar que el cliente ya esté creado y activo en PeYA.
    • Verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de iniciar la integración.
    • Si el cliente está asociado a otro POS, solicitar que realice la baja antes de proceder con nuestra integración.
    • Asegurar que el cliente tenga los datos necesarios para la integración con PeYA:
      • ID de PeYA.
      • Precios por lista para PeYA
      • Validación si es tienda única o si se encuentra vinculada ( esto es similar a la estructura Padre e Hija de Rappi).
    • Confirmar con el cliente que los productos tengan imágenes (Mínimo el 80% de los productos)
    • Validar estructura del menú de acuerdo a lo que el cliente requiere y verificar que se cumplan con las restricciones que PeYA exige para la recepción del menú.
    • Programar la integración para inicios de semana e informar al cliente sobre el tiempo de activación de 1 a 3 días hábiles.

5. Procedimiento

  • Paso 1: Solicitud de Datos vía correo.
    • El agente encargado de realizar la activación de la integración PeYA- Toteat debe solicitar los datos de configuración vía correo electrónico.
    • Se requiere la siguiente información:
      1.- ID de PeYA.

      2.- Validación si es tienda única, o se encuentra vinculada a otra.

      3.- Validar estructura del menú de acuerdo a lo que el cliente requiere.

      4.- Precios por lista a usar en PeYA, para ello se debe crear el precio por lista en el ambiente e identificarlo con el nombre de la marca-PeYA, posteriormente, se debe descargar el archivo de precio x lista en formato de Excel y enviarlo al cliente.

      -En la columna de precio por lista, el cliente deberá colocar los precios a manejar en dicha plataforma.

      -Si no desea que un producto sea visible en PeYA, deberá dejar la celda en blanco (sin datos).

      -Este proceso debe realizarse tanto para los productos como para los extras, asegurándonos de que la información sea completa y precisa para una correcta implementación en la plataforma de PeYA.
    • Nota: El cliente debe responder a lo solicitado a través del mismo correo, con el fin de mantener un registro formal de los datos.
  • Paso 2: Solicitud del Remote Id a Peya.
    • Es importante destacar que en el año 2024 Peya implementó un cambio en su integración, específicamente en su API. La integración anterior utilizaba un SDK, mientras que ahora se emplea la nueva API V2. Para esta integración, sigue siendo necesario el ID de Peya para identificar la tienda. Sin embargo, para la configuración en la API de Toteat se requiere el "Remote id", un dato proporcionado por Peya tras enviar la solicitud correspondiente de activación, indicando el ID del restaurante de Peya.
    • Esta solicitud del Remote Id se realiza por medio del correo de Soporte de Peya:
      mailto:soporteintegraciones@pedidosya.com
    • La solicitud para Argentina se está manejando aun por el grupo de Whatsapp “Integración Toteat-PYA AR”
  • Paso 3: Validación si es tienda única o vinculada.

    En este caso, si el cliente no está seguro como está configurada su restaurante en Peya, existen varias formas de obtener esa información:

    1.- Realizar la consulta a la Kam de PeYa.

    2.- Realizar la consulta a Soporte de PeYa vía correo:
    mailto:soporteintegraciones@pedidosya.com

    • En el caso de Argentina se se puede realizar la consulta por el grupo de Whatsapp “Integración Toteat-PYA AR”

6. Salida

  • Disponer de los datos de configuración para la integración de PeYA es fundamental para agilizar y garantizar una ejecución correcta. Esta información permite realizar el proceso rápidamente, asegurando que se cumplan los parámetros establecidos. Así, no sólo se optimiza el tiempo empleado en la activación de la integración, sino que también se asegura la precisión y fiabilidad del proceso.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de PeYA.

8. Puntos de control

  • Revisión del proceso de obtención de datos de PeYA.
  • Validación continua del flujo para la solicitud de datos, con el área de Integraciones PeYA.

8.1. Datos para configuración Didi

PropietarioFFrancisco Ramirez
Etiquetas

1. Descripción

En este módulo, abordaremos los campos de configuración más importantes durante la activación de la integración de Toteat con Didi.

2. Alcance

Para comenzar la integración con Didi, es fundamental comprender las configuraciones necesarias que asegurarán un proceso exitoso. Las configuraciones de las APIs incluyen campos estándar que siempre deben configurarse y en el caso de Didi, es crucial validar que se genere el URL de conexión en la API una vez que se hayan guardado los ajustes.

3. Entrada

Cuando el cliente desea iniciar su operación con la integración de Didi, se inicia el proceso de configuración, la integración de Didi es uno de los procesos más sencillos y rápidos, solo se debe tener identificado los Id restaurante -Id local de Toteat y proceder a configurar la Api.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integraciones y tener conocimientos sólidos sobre el proceso de activación en la integración Didi-Toteat.
    • Confirmar que el cliente ya esté creado y activo en Didi.
    • Verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de iniciar la integración. Si el cliente está asociado a otro POS, debe desvincular su tienda del otro POS.
    • Asegurar que el cliente tenga los datos necesarios para la integración: Precios por lista para Didi y acceso a la plataforma de Didi.
    • Confirmar con el cliente que los productos tengan imágenes.
    • Tener a mano los datos de Toteat.
    • Programar la integración para inicios de semana e informar al cliente que el tiempo de activación dependerá de su disposición, ya que la vinculación estará 100% sujeta a su gestión en el portal de Didi.

5. Procedimiento

  • Paso 1: Datos del Api de Toteat.
    • Ver Módulo 📗2.2. ¿Cómo se configura un API? (Excluir paso 5)
      • Una vez configurada la Api de DIDI y grabar los ajustes indicados en el punto anterior, se observará una URL en la sección General de la API, esta URL será usada en el siguiente paso.

  • Paso 2: Indicaciones vía correo
    • El agente encargado de realizar la activación de la integración Didi-Toteat debe indicar al cliente vía correo como proceder para la integración Didi-Toteat, el cliente deberá ingresar a su portal de DIDI y autorizar la activación a través del enlace que se le enviará vía correo, este enlace es la URL que la Api generó (Paso 1).

6. Salida

  • Para la integración de Didi, es fundamental indicar al cliente cuáles son los datos necesarios para iniciar la configuración de la integración. En el caso de Didi, es un proceso más sencillo, ya que los datos que se requieren son la URL que se genera en la API y las credenciales para ingresar al portal de DIDI.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de DIDI.

8. Puntos de control

  • Validación de creación del API y generación de la URL

1. Descripción

Una vez recopilados los datos de configuración para la integración entre Toteat y la plataforma de Justo, el siguiente paso es iniciar el proceso de activación. Esto implica validar con el cliente los datos suministrados en Justo y los generados en Toteat y configurar la API correspondiente para establecer la conexión entre ambas partes.

2. Alcance

Al activar la integración entre Justo y Toteat, nuestros clientes podrán disfrutar de las funcionalidades principales que ofrece esta integración. Entre ellas se encuentran la sincronización de menús, la gestión de pedidos y la generación de reportes consolidados.

3. Entrada

Para iniciar el proceso de activación, el cliente debe realizar la solicitud a su agente de Onboarding o de Customer, según el estatus del restaurante en Toteat.

  • En el caso de un cliente nuevo, el agente de Onboarding debe validar durante el proceso si se llevará a cabo la activación de la integración. Es importante destacar que esta información debería haber sido comunicada por el cliente al comercial durante la negociación, para que el servicio quede especificado en Salesforce.
  • Si el cliente ya está en cartera, debe enviar la solicitud al correo clientes@toteat.com, donde recibirá una respuesta con los pasos a seguir.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integración y tener conocimientos sólidos sobre el proceso de activación en la integración de Justo-Toteat.
    • Confirmar con el cliente que ya está creado y activo en Justo, además de contar con su autorización para iniciar el proceso.
    • Es crucial verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de comenzar el proceso de integración.
    • El cliente debe contar con los datos necesarios para la integración, que incluyen: el Api Key, creación de Webhooks en Justo, los precios por lista que quiere mostrar en Justo.
    • Validar con el cliente que los productos tengan imágenes.
    • Verificar tener los datos de configuración a la mano, estos datos son fundamentales para garantizar un proceso de integración exitoso y sin contratiempos. Si tiene alguna pregunta o requiere asistencia adicional se debe agendar una reunión vía meet con el cliente y verificar todas las dudas sobre el procedimiento.. Ver
    • Si el restaurante tiene múltiples sedes, es importante asegurarse de que todos los menús estén estandarizados, preferiblemente utilizando Toteat en todas las sedes.
    • Programar la integración para inicios de semana e informar al cliente que el tiempo de activación es de 1 a 3 días hábiles.

    • Id Menu Externo: En este campo se coloca el Id del local generado en Justo, dato suministrado por el cliente o se puede consultar en versiones por medio del api Key, ver 2.- Los datos de Id de Menú externo y Id externa restaurantes lo puede indicar el cliente o se puede consultar por versiones por medio del Api key.
  • Paso 2: Activación Api en Toteat.
    • Una vez que hayamos confirmado que todas las configuraciones de la API de Justo son correctas, procederemos a activar la API y dar clic al botón Grabar.
  • Paso 3: Validación de configuraciones en el Admin de Justo.
    • La activación de la integración Justo - Toteat es un proceso que depende de que en ambas partes las configuraciones estén en los campos correctos, por lo tanto es obligatorio validar con el cliente, ya sea a través de una videollamada o mediante capturas de pantalla, las cuales se pueden enviar por correo, poder verificar que las configuraciones en Justo estén correctas.
    • Los campos que debemos validar en Justo son los siguientes:

      Validar Api Key: Verificar que las últimas letras del api Key configurado en Justo para Toteat sean iguales a las que están en la API de Toteat.

      Validar Webhooks: Los webhooks serán identificados con la URL enviada de Toteat
      Ver

      • Al ingresar al Webhook debe estar indicado el local al que pertenece y activo.

  • Paso 4: Validación de status de la integración en Versiones.
    • Es necesario validar status de la integración en versiones luego de haber recibido realizado las activacion.
      • Al ingresar a Versiones, en la sección de Delivery, selecciona Justo y luego utiliza el buscador para colocar el ID del restaurante.
        • Validar que esta activa: Verificar que las marcas y APIs que se integraron estén activas en la primera columna "Activa".
          En este campo se valida la activación de parte de Toteat.

6. Salida

  • La integración con Justo es crucial para optimizar la experiencia tanto del restaurante como de la gestión de la plataforma de delivery. Proporciona una mayor eficiencia en el proceso y facilita la centralización de la operación. Por lo tanto, es de gran importancia comprender el proceso de activación de la integración con Justo para garantizar una activación eficiente y una operación fluida. Además, es fundamental tener en cuenta que Justo centraliza la información de las integraciones de las sedes de una marca en una página, por lo que las cartas deben estar estandarizadas tanto en código como en estructura.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de Justo.

8. Puntos de control

  • Auditar el proceso de activación de la integración de Justo en todos los países donde aplique.
  • Validar status de la integración.

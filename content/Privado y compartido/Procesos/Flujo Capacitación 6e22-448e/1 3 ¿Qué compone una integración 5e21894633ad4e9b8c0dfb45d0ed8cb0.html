1. Descripción

En este módulo entenderemos cómo está compuesta una integración.

2. Alcance

La composición de una integración es fundamental para identificar los elementos claves que intervienen en su correcto funcionamiento. Esto incluye la configuración de los campos necesarios, los datos que deben ser proporcionados por los clientes y los flujos de operación involucrados.

3. Entrada

Personal de Toteat enfocado en la activación de procesos, manejo del cliente y solución de incidencias.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe cumpplir con los siguientes perfiles:
    • Personal enfocado en la atención al cliente y la activación de procesos.
    • Pertenecer al equipo Toteat.
    • Tener conocimientos del software Toteat.

5. Procedimiento

  • Paso 1: POS

    POS (Point of Sale) gastronómico, también conocido como "sistema restaurantero" o "software gastronómico", es el software usado para la gestión de negocios relacionados con el restaurante, en nuestro caso Toteat.

  • Paso 2: Plataforma de Delivery o Ecommerce.

    La plataforma de delivery y/o Ecommerce son servicios en línea que permite a los usuarios pedir comida o productos de diversos restaurantes y comercios locales a través de una aplicación móvil o un sitio web. Estas plataformas actúan como intermediarias entre los negocios y los consumidores, facilitando el proceso de pedido, pago y entrega.

    Toteat se integra a las siguientes plataformas de Delivery:

    • UberEats
    • PedidosYA
    • Rappi
    • Justo
    • Didi

    Algunos de los ecommerce que hoy en dia se integran a Toteat son:

    • Mercat
    • Cluvi
    • Tucan
    • Hubster (Otter)
    • Agil
    • Otros.

    Nota: Cada plataforma posee un código (Id) que identifica al restaurante.

  • Paso 3: Api

    Una API (interfaz de programación de aplicaciones) es un conjunto de reglas y protocolos que facilita la comunicación entre diferentes aplicaciones o sistemas. Actúa como un puente que permite la integración y la operación automatizada entre el punto de venta (POS) y la plataforma de delivery.

    Nota: Las APIs en Toteat son muy similares, pero dependiendo de la plataforma a integrar manejan configuraciones particulares.

  • Paso 4: Webhooks

    Los webhooks son los que permiten enviar notificaciones push a una URL seleccionada cada vez que ocurre una actividad específica, permitiendo la integración y la transferencia de datos en tiempo real entre aplicaciones.

  • Paso 5: Tokens

    Son códigos alfanuméricos generados para autenticar la comunicación y así poder acceder a cuentas o sistemas.

  • Paso 6: Id externos

    Es un identificador único asociado al restaurante en la plataforma de delivery. Se utiliza para vincular el restaurante al configurarlo en la API, permitiendo así la integración y el intercambio de datos entre diferentes sistemas.

6. Salida

  • Al conocer la composición de las integraciones, se permite validar las configuraciones, analizar posibles problemas y detectar errores de manera precisa. Este proceso nos permite determinar las soluciones o flujos a aplicar entre las áreas implicadas, garantizando así un funcionamiento óptimo de la integración. Al comprender a fondo cada uno de estos aspectos, podemos asegurar que la integración cumpla con los requisitos necesarios y opere de manera eficiente, brindando una experiencia satisfactoria tanto para los clientes como para el equipo encargado de su implementación.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.

8. Puntos de control

  • Evaluación de homologación de conocimientos.
  • Capacitaciones.
  • Activaciones de integraciones.

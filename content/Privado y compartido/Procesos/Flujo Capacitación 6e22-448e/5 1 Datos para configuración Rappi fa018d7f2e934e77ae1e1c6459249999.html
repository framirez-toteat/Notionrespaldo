1. Descripción

En este módulo, abordaremos los campos de configuración más importantes durante la activación de la integración de Toteat con Rappi.

2. Alcance

Para iniciar el proceso de integración con Rappi, es crucial comprender las configuraciones necesarias para garantizar un proceso de integración exitoso.
Las configuraciones de las APIs incluyen campos estándar que siempre deben configurarse, así como campos específicos que requieren datos exclusivos de Rappi.

3. Entrada

Cuando el cliente desea iniciar su operación con la integración de Rappi, comienza este proceso de solicitud y recopilación de datos para las configuraciones para llevar a cabo la integración con Rappi.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integraciones y tener conocimientos sólidos sobre el proceso de activación en la integración Rappi-Toteat.
    • Confirmar que el cliente ya esté creado y activo en Rappi.
    • Verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de iniciar la integración.
    • Si el cliente está asociado a otro POS, solicitar que realice la baja antes de proceder con nuestra integración.
    • Asegurar que el cliente tenga los datos necesarios para la integración con Rappi:
      • Store ID de Rappi.
      • Precios por lista para Rappi.
      • Validación si es tienda única, Padre o Hija.
    • Confirmar con el cliente que los productos tengan imágenes (Mínimo el 80% de los productos)
    • Validar estructura del menú de acuerdo a lo que el cliente requiere y verificar que se cumplan con las restricciones que Rappi exige para la recepción del menú.
    • Programar la integración para inicios de semana e informar al cliente sobre el tiempo de activación de 1 a 3 días hábiles.

5. Procedimiento

  • Paso 1: Solicitud de Datos vía correo
    • El agente encargado de realizar la activación de la integración Rappi- Toteat debe solicitar los datos de configuración vía correo electrónico.
    • Se requiere la siguiente información:
      1.- Store ID de Rappi.

      2.- Validación si es tienda única, Padre o Hija.

      3.- Validar estructura del menú de acuerdo a lo que el cliente requiere.

      4.- Precios por lista a usar en Rappi, para ello se debe crear el precio por lista en el ambiente e identificarlo con el nombre de la marca-Rappi, posteriormente, se debe descargar en formato de Excel, el archivo de precio por lista y enviarlo al cliente.

      -En la columna de precio por lista, el cliente deberá colocar los precios a manejar en dicha plataforma.

      -Si no desea que un producto sea visible en Rappi, deberá dejar la celda en blanco (sin datos).

      -Este proceso debe realizarse tanto para los productos como para los extras, asegurándonos de que la información sea completa y precisa para una correcta implementación en la plataforma de Rappi.
    • Nota: El cliente debe responder a lo solicitado a través del mismo correo, con el fin de mantener un registro formal de los datos.
  • Paso 2: Ubicación del Store id en la plataforma de Rappi.
    • Ingresar al portal Rappi Aliados con las credenciales que Rappi te otorgo.
    • Despliega el menú en la barra lateral izquierda.
    • Selecciona la opción “Mi cuenta”
    • Ubica la sección de mis tiendas. Una vez allí, podrás ubicar el store id correspondiente.
    • Nota: Esta tarea debe ser realizada por el cliente. Es importante que el cliente verifique que está en el portal Rappialiados correcto, especialmente si posee varias marcas. Este paso es crucial para asegurar que se esté trabajando con la marca adecuada y se eviten errores en el proceso.
  • Paso 3: Validación si es tienda única, Padre o Hija.

    En este caso, si el cliente no está seguro del tipo de tienda que es su restaurante en Rappi, e existen varias formas de obtener esa información:

    1.- Realizar la consulta a la Kam de Rappi.

    2.- Realizar la consulta a Soporte de Rappi: http://wa.me/5715140348

    3.- Realizar la validación en el grupo de Whatsapp que se tiene con rappi para las activaciones.

6. Salida

  • Disponer de los datos de configuración para la integración de Rappi es fundamental para agilizar y garantizar una ejecución correcta. Esta información permite realizar el proceso rápidamente, asegurando que se cumplan los parámetros establecidos. Así, no sólo se optimiza el tiempo empleado en la activación de la integración, sino que también se asegura la precisión y fiabilidad del proceso.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de Rappi.

8. Puntos de control

  • Revisión del proceso de obtención de datos de Rappi.

1. Descripción

En este módulo, abordaremos los campos de configuración más importantes durante la activación de la integración de Toteat con Uber.

2. Alcance

Para iniciar el proceso de integración con Uber, es crucial comprender las configuraciones necesarias que garantizará un proceso exitoso. Las configuraciones de las APIs incluyen campos estándar que siempre deben configurarse, así como campos específicos que requieren datos exclusivos para Uber.

3. Entrada

Cuando el cliente desea iniciar su operación con la integración de Uber, comienza este proceso de solicitud y recopilación de datos para las configuraciones de Uber.

4. Condiciones

  • La persona que realice y/o solicite este proceso debe tener en consideración:
    • Ser personal autorizado de Toteat para la activación de los procesos de integraciones y tener conocimientos sólidos sobre el proceso de activación en la integración Uber-Toteat.
    • Confirmar que el cliente ya esté creado y activo en Uber Eats.
    • Verificar que el cliente no esté vinculado a otro punto de venta (POS) antes de iniciar la integración.
    • Si el cliente está asociado a otro POS, solicitar que realice la baja antes de proceder con nuestra integración.
    • Asegurar que el cliente tenga los datos necesarios para la integración: UUID, precios por lista para Uber Eats y turnos a utilizar en la plataforma.
    • Confirmar con el cliente que los productos tengan imágenes.
    • Tener a mano los datos de Toteat en Uber Eats y el formulario correspondiente.
    • Programar la integración para inicios de semana e informar al cliente sobre el tiempo de activación de 1 a 3 días hábiles.

5. Procedimiento

  • Paso 1: Solicitud de Datos vía correo
    • El agente encargado de realizar la activación de la integración Uber-Toteat debe solicitar los datos de configuración por correo electrónico.
    • Se requiere la siguiente información:
      1.- UUID de Uber (Ver Paso 2)
      2.- Turnos a manejar en Uber.
      3.- Precios por lista a usar en Uber, para ello se debe crear el precio por lista en el ambiente e identificarlo con el nombre de la marca-Uber, posteriormente, se debe descargar en formato de Excel, el archivo de precio por lista y enviarlo al cliente.
      En la columna de precio por lista, el cliente deberá colocar los precios a manejar en dicha plataforma. Si no desea que un producto sea visible en Uber, deberá dejar la celda en blanco (sin datos).
      Este proceso debe realizarse tanto para los productos como para los extras, asegurándonos de que la información sea completa y precisa para una correcta implementación en la plataforma de Uber.
    • Nota: El cliente debe responder a lo solicitado a través del mismo correo, con el fin de mantener un registro formal de los datos.

6. Salida

  • Disponer de los datos de configuración para la integración de Uber es fundamental para agilizar y garantizar una ejecución correcta. Esta información permite realizar el proceso rápidamente, asegurando que se cumplan los parámetros establecidos. Así, no sólo se optimiza el tiempo empleado en la activación de la integración, sino que también se asegura la precisión y fiabilidad del proceso.

7. Excepciones del flujo

  • Toda integración que no corresponda a Delivery.
  • Restaurantes que aún no se encuentren vinculados en la plataforma de Uber.
